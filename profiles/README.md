# Profiles

Profiles specialize the core workflow without forking it. They are deliberately conservative in this first release.

- `domains/` defines domain-specific evidence, experiment, safety, and reproducibility requirements.
- `venues/` defines current submission, review, artifact, and formatting expectations.

A profile is active only when the project state names it. Verify time-sensitive venue rules from official sources before use. Project decisions may strengthen a profile; they must not silently weaken provenance, citation truthfulness, safety, or human release gates.
